## What I build

- Developer tooling that removes friction without lowering standards.
- AI-assisted workflows shaped by context, rules, and verification.
- Web products with a strong bias toward clarity, maintainability, and delivery quality.

## How I work

- I trust architecture and checks more than clever prompting.
- I prefer fewer moving parts, then add structure only when the system earns it.
- I document tradeoffs, failures, and reversals instead of polishing them away.
- I care about tools that help people stay in control when systems get more automated.
